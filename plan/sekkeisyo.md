# 音楽・音声ファイル転送アプリ

## 概要

* ローカルコンピューターとファイル転送モードで接続した Android スマートフォンとの間で、音楽ファイルを容易に転送できるようにする Windows GUIアプリ

## 不要な機能

* **自動同期機能は不要**
  * あくまでもユーザー操作による転送であること
* アプリ自体による再生機能
  * 曲のダブルクリックor右クリックメニューからデフォルトの再生ソフトを呼ぶのはあり
* ファイルタイプ、エンコーディングの変換機能

## 必要な機能

* 指定したローカルファイルシステムのフォルダ内にある音楽ファイルのライブラリ作成、表示機能
  * ライブラリ構築、更新は初回とユーザー任意で再構築指定時（差分でもよい）
  * 表示は以下のものを自由に切り替えられる
    * 全ての曲一覧
      * リスト表示
    * アーティスト一覧 → ダブルクリックでアーティストの曲一覧
      * アーティスト、アーティストの曲一覧はリスト表示
    * アルバム一覧 → ダブルクリックでアルバム内の曲一覧
      * アルバム一覧はジャケット画像を並べて表示できること
        * ジャケット画像の下部にアルバム名とアルバムアーティスト名が記載されるイメージ
      * アルバム内の曲一覧はリスト表示
  * ダブルクリック、もしくは右クリックメニューからOSデフォルトの再生ソフトを呼び出して再生する機能
  * 曲の右クリックメニューから各音楽ファイルのメタデータを取得、閲覧できる機能
    * 現時点で編集機能は不要
* 各一覧からファイルを選択してスマートフォンの指定のディレクトリにコピーする機能
  * 曲、アーティスト、アルバム のいずれかを GUI の特定領域（スマートフォンであることを記載）にドラッグ＆ドロップするような操作
  * ファイルはそのままコピーする
  * スマートフォン側の配置フォルダは指定されたフォルダ以下の `<アーティスト名>/<アルバム名>/ファイル` となるようにする
  * アーティスト、アルバムを指定した場合はその中に含まれる全ての音楽ファイルをコピーする
  * **重複するファイルはコピーしない**
    * 上書きや別名保存はせず、スキップする

## 音楽ファイルとして指定するもの

* 一般的に音楽ファイルとされているものは全部対象とする
* `.mp4` も対象
  * 動画ファイルと同じ拡張子だが、メタデータが取得出来れば音楽ファイルとして扱ってよい

## 実装環境

* 以下の条件をもとに最適なものを選択する
  * Windows GUI で構築できるもの
  * 基本的に無料で環境が構築できるもの
  * 制作物自体の商用利用も考慮しない
  * 実行ファイル単独で動くものが理想だが、インストーラーがあってもよい
* 事前にインストールが必要なコンポーネント類は PLAN 時にリストアップすること
  * Agent によるインストールがかのうなものと、こちらでの手動インストールとアクティベートが必要なものをそれぞれ示すこと

## 注意事項

* ライブラリに入る音楽ファイル数は概算で5000程度
  * 10000 程度までは問題なく動作するのが望ましい
  * ライブラリ構築には多少時間がかかってもよい
